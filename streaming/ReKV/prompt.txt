WHAT THIS PROJECT DOES
-----------------------
Evaluate 6 streaming video-QA methods on RVS benchmark (ego + movie videos).
Each method ingests one sampled frame at a time and answers open-ended questions
about what it saw — no offline whole-video prefill.
Goal: compare quality (judge score + ROUGE-L F1), answer latency, and GPU/CPU memory.
Models evaluated: LLaVA-OV 0.5B (ReKV methods 1-4 complete; StreamingTom 5-6 pending full eval).

1. full_streaming    — plain causal KV cache; all frames kept on GPU
                       KV cache grows to ~9 GB (movie) / ~13 GB (ego 60-min)

2. duo_streaming     — Duo head routing: ~25% full-attn heads (all tokens),
                       ~75% streaming heads (sink+recent window only)
                       requires block_sparse_attn kernel; window: sink=256 recent=512

3. rekv              — local window on GPU + CPU offload + top-k cosine retrieval at Q time
                       ~3.2 GB GPU + ~2.3 GB CPU (movie) / ~3.1 GB GPU + ~4 GB CPU (ego)

4. duo_plus_rekv     — ReKV retrieval then Duo head routing over assembled context
                       ~3.2 GB GPU + ~2.3 GB CPU; approximate hybrid

5. streamingtom      — Native StreamingTom: CTR per-frame token compression (50 tok/frame)
                       + OQM 4-bit quantized memory with online retrieval
                       ~15.7x KV cache compression; uses lmms-lab LLaVA-OV variant

6. duo_plus_streamingtom — StreamingTom runtime + Duo head routing inside Qwen2 forward
                       True hybrid: full heads attend full KV, streaming heads attend
                       sink+recent window inside StreamingTom attention forward

DATA FLOW
----------
datasets.py              load RVS annotation JSON, sort convs by end_time, sample
                         frames at sample_fps, resolve video path
  ↓
[optional] precompute_features.py → feature_cache.py
                         pre-extract visual tokens; ReKV methods only
                         StreamingTom uses raw frames always (ingest_features is a stub)
  ↓
methods 1-4 (methods.py)  per-video reset; causal ingest (one frame per forward pass)
  full_streaming:   flash_attention_2 KV cache
  duo_streaming:    block_sparse_attn for streaming heads
  rekv:             patch_hf → rekv_attention_forward → ContextManager
  duo_plus_rekv:    same + Duo weights → rekv_duo_enabled=True

methods 5-6 (StreamingTom/scripts/eval/run_eval.py)
  streamingtom:              generate_with_streamingtom_streaming(images=frame)
  duo_plus_streamingtom:     same + _hybrid_duo_streamingtom_attention at each layer
  ↓
answer_question → greedy_decode_with_cache
  retrieval triggered for rekv/duo_plus_rekv (cosine-sim CPU→GPU)
  OQM dequantisation for streamingtom methods
  ↓
run_eval.py (ReKV) / run_eval.py (ST)
  write result JSON, checkpoint/resume, sharding
  both use identical normalize_result_payload_schema from streaming.ReKV.run_eval

OUTPUT STRUCTURE
-----------------
outputs/evaluations_streaming/<dataset>/
  full_eval/
    run1/<method>/chunk_*.json               ReKV methods (existing complete results)
    run2/<method>/chunk_*.json               StreamingTom methods (new)
    merged_all/
      merged/<method>.json                   merged for all 6 methods
      comparison/summary.md                  headline quality/latency/memory table
      comparison/*.png                       stability plots
      plots/*.png                            ROUGE-L based (15 PNGs)
      plots_judge/*.png                      judge-score based (after --run-judge)

Result JSON top-level keys (all 6 methods — same schema):
  run_config              CLI args used
  evaluation_manifest     model, sample_fps, backend, rekv/duo/st config
  aggregate_metrics       avg_rouge_l_f1, avg_judge_score, avg_answer_latency_sec,
                          peak_memory_bytes, peak_cpu_offload_bytes (null for ST),
                          avg_retrieval_latency_sec (null for full_streaming/duo)
  videos[]
    conversations[].prediction       model's generated answer
    conversations[].reference_answer ground truth
    conversations[].scores.rouge_l_f1
    conversations[].method_stats     timing, retrieval, peak_memory_bytes,
                                     cpu_offload_bytes_peak, avg_retrieved_block_count
