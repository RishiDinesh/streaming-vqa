Streaming ReKV handoff
======================

Fast context for the streaming/ReKV module on the exp/nv-gpu-inference branch.

Branch / environment
--------------------
- Branch:   exp/nv-gpu-inference
- Cluster:  NVIDIA SLURM
- Conda env: <repo-root>/envs/duo  (created by setup.sh)
- Activate: conda activate <repo-root>/envs/duo

Datasets
--------
- rvs_ego:   annotation ego/ego4d_oe.json,     HF subset ego
- rvs_movie: annotation movie/movienet_oe.json, HF subset movie
- HF repo: Becomebright/RVS

Methods (all four)
------------------
- full_streaming   — plain causal KV cache; no compression
- duo_streaming    — Duo head routing; blocksparse kernel required
- rekv             — local window on GPU + CPU offload + top-k retrieval at question time
- duo_plus_rekv    — ReKV retrieval then Duo head routing over assembled context (approximate)

Streaming contract (enforced identically across all methods)
------------------------------------------------------------
- Causal ingest only: one sampled frame per forward pass
- Visibility cutoff: sampled timestamps strictly < conversation end_time (bisect_left)
- Conversations sorted by end_time at load time
- Shared KV state across all questions from the same video
- No offline whole-video prefill
- Feature cache must exactly match the run's sampling schedule

Key files
---------
- streaming/ReKV/run_eval.py              — main eval loop (causal ingest, resume, sharding)
- streaming/ReKV/methods.py               — all four method classes, shared utilities

- streaming/ReKV/validate_runtime_env.py  — report actual backend stack

- streaming/ReKV/profile_streaming.py     — single-video latency/memory profiling
- streaming/ReKV/compare_subsamples.py    — cross-slice stability analysis
- streaming/ReKV/smoke_test.py            — fast unit tests (no GPU required)

- streaming/ReKV/rekv_core/attention/rekv_attention.py   — ReKV attention forward

Key scripts
-----------
- scripts/streaming_env.sh                    — conda env activation (project-local first)
- streaming/ReKV/run_eval.sh                  — single-GPU SLURM eval job
- streaming/ReKV/profile_streaming.sh         — SLURM profiling job
- scripts/run_streaming_eval_slurm_array.sh   — multi-GPU sharded SLURM array
- scripts/run_streaming_subset3_slurm.sh      — 3-video subset: all 4 methods, 4 jobs

Workflow
--------
1. Setup (once):
     bash setup.sh

2. Validate backend:
     python -m streaming.ReKV.validate_runtime_env
     # Expect: streaming_attn_backend_actual = blocksparse

3. Run 3-video subset (all 4 methods):
     bash scripts/run_streaming_subset3_slurm.sh \
       --annotation-path /path/to/ego4d_oe.json \
       --video-root /path/to/videos \
       --dataset rvs_ego \
       --attn-dir /path/to/duo/weights

4. Full eval (single GPU):
     sbatch --output="$(pwd)/logs/%x-%j.out" streaming/ReKV/run_eval.sh \
       --dataset rvs_ego --method rekv

5. Full eval (multi-GPU sharded):
     NUM_CHUNKS=8 DATASET=rvs_ego METHOD=rekv \
       sbatch --array=0-7 \
         --output="$(pwd)/logs/%x-%A_%a.out" \
         scripts/run_streaming_eval_slurm_array.sh

6. Judge answers:
     python -m streaming.ReKV.judge_results --in-place <results.json>

7. Plot:
     python -m streaming.ReKV.plot_results <results.json> --output-dir outputs/plots/

Backend requirements
--------------------
- flash-attn:        full-attention forward (strongly recommended)
- block_sparse_attn: sparse Duo streaming (required for paper-faithful Duo)
- flashinfer:        RMSNorm/RoPE acceleration (optional)
- decord:            fast video decoding (recommended)
Block-Sparse install: cd Block-Sparse-Attention && BLOCK_SPARSE_ATTN_CUDA_ARCHS="80;89;90" python setup.py install

Result categories in manifests
-------------------------------
- nvidia_sparse_duo_equivalent  — block_sparse_attn active, paper-faithful
- sdpa_fallback_duo             — block_sparse_attn missing (should not occur on NVIDIA)
