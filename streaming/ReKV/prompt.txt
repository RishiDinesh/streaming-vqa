Methods (all four)
------------------
- full_streaming   — plain causal KV cache; no compression
- duo_streaming    — Duo head routing; blocksparse kernel required
- rekv             — local window on GPU + CPU offload + top-k retrieval at question time
- duo_plus_rekv    — ReKV retrieval then Duo head routing over assembled context (approximate)

Streaming contract (enforced identically across all methods)
------------------------------------------------------------
- Causal ingest only: one sampled frame per forward pass
- Visibility cutoff: sampled timestamps strictly < conversation end_time (bisect_left)
- Conversations sorted by end_time at load time
- Shared KV state across all questions from the same video
- No offline whole-video prefill
- Feature cache must exactly match the run's sampling schedule

First-time setup (do once)
--------------------------
1. Install Miniconda (login node):
     bash /tmp/Miniconda3-latest-Linux-x86_64.sh -b -p /u/navdeep/miniconda3
     source /u/navdeep/miniconda3/etc/profile.d/conda.sh

2. Build env (must be on a GPU compute node — compiles CUDA extensions):
     srun --nodes=1 --ntasks=1 --gres=gpu:1 --cpus-per-task=8 --mem=32G \
          --time=01:00:00 --pty bash -l
     # then inside the compute node:
     source /u/navdeep/miniconda3/etc/profile.d/conda.sh
     cd /w/nobackup/385/scratch-space/expires-2026-Apr-23/navy/streaming-vqa
     BLOCK_SPARSE_ATTN_CUDA_ARCHS="80;89;90" bash setup.sh 2>&1 | tee logs/setup_$(date +%Y%m%d_%H%M%S).log

3. Validate (still on compute node):
     conda activate <repo-root>/envs/duo
     python -m streaming.ReKV.validate_runtime_env
     # Expect: "streaming_attn_backend_actual": "blocksparse"

BLOCK_SPARSE_ATTN_CUDA_ARCHS: A100=80, H100=90, L40/Ada=89. Default "80;89;90" covers all.
Check GPU arch: nvidia-smi --query-gpu=name,compute_cap --format=csv,noheader

Workflow
--------
1. Smoke test (1 video, all 4 methods — verify env works):
     bash scripts/run_streaming_subset3_slurm.sh --max-videos 1

2. 5-video subset:
     bash scripts/run_streaming_subset3_slurm.sh --max-videos 5

3. Full eval (single GPU):
     sbatch --output="$(pwd)/logs/%x-%j.out" streaming/ReKV/run_eval.sh \
       --dataset rvs_ego --method rekv

4. Full eval (multi-GPU sharded):
     NUM_CHUNKS=8 DATASET=rvs_ego METHOD=rekv \
       sbatch --array=0-7 \
         --output="$(pwd)/logs/%x-%A_%a.out" \
         scripts/run_streaming_eval_slurm_array.sh

5. After jobs finish (script prints exact commands):
     python -m streaming.ReKV.compare_subsamples \
         <result1.json> <result2.json> <result3.json> <result4.json> \
         --output-dir outputs/evaluations_streaming/rvs-ego/subset1/comparison/
     python -m streaming.ReKV.plot_results \
         <result1.json> <result2.json> <result3.json> <result4.json> \
         --output-dir outputs/evaluations_streaming/rvs-ego/subset1/plots/
